# 跨境电商建站 SaaS 需求说明

## 1. 文档说明

本文档用于重定义当前系统的产品需求基线。

新需求目标：
- 系统主要服务于跨境电商建站用户
- 核心交付不是内容发布，而是“建站 + 斗篷 + ERP + 支付”
- 后端管理页面必须支持编辑网站布局

本文档用于指导后续的产品设计、数据建模、接口设计和开发排期。

---

## 2. 产品目标

平台要解决的问题：
- 跨境卖家需要快速搭建多个站点，但建站成本高、上线慢
- 运营团队需要在一个后台统一管理多个站点的页面、域名、支付和订单
- 投放团队需要按渠道和流量来源做斗篷分流
- 商家需要基础 ERP 能力管理商品、库存、订单和发货

产品目标：
- 让用户可以在几分钟内完成一键建站
- 让非技术运营可以通过后台编辑页面布局
- 让站点能接入斗篷规则、支付通道和订单履约流程
- 让多个站点可以复用模板、支付、商品和运营配置

---

## 3. 目标用户

## 3.1 商户主账号

典型人群：
- 跨境电商老板
- 站群运营负责人
- 投放负责人

核心诉求：
- 快速开新站
- 批量复制站点
- 配置支付与域名
- 查看订单与站点状态

## 3.2 运营人员

核心诉求：
- 编辑页面布局
- 上下架商品
- 配置斗篷策略
- 查看订单与支付结果

## 3.3 客服 / 发货人员

核心诉求：
- 查看订单
- 更新发货状态
- 处理异常订单

## 3.4 平台超级管理员

核心诉求：
- 管理租户、套餐、支付通道、模板、全局配置
- 审核风险站点
- 监控系统运行状态

---

## 4. 角色与权限

角色建议：
- 平台超级管理员
- 商户主账号
- 站点管理员
- 运营
- 客服
- 财务

权限控制要求：
- 按租户隔离
- 按站点授权
- 按模块授权
- 关键操作必须审计

关键高风险操作：
- 删除站点
- 发布页面版本
- 修改斗篷规则
- 修改支付通道
- 手工改订单状态

---

## 5. 核心业务场景

## 5.1 一键建站

用户希望通过选择模板和填写少量信息，快速生成可运营站点。

预期效果：
- 自动生成站点
- 自动初始化首页、商品页、结账页、成功页
- 自动配置基础主题和默认区块
- 可立即进入页面编辑器继续装修

## 5.2 页面装修

用户希望在后台可视化编辑页面布局，而不是进入独立的 WordPress 后台逐页修改。

预期效果：
- 页面区块可增删改排
- 颜色、字体、按钮、商品区、信任区块可配置
- 支持预览与发布
- 支持模板复用

## 5.3 斗篷配置

用户希望针对不同流量来源展示不同页面版本，控制审核流量和正常流量。

预期效果：
- 可配置多条件规则
- 可查看命中日志
- 可手动停用或灰度发布

## 5.4 商品与订单管理

用户希望在一个后台维护商品、订单和库存，并与支付、发货流程串联。

预期效果：
- 商品管理
- 订单查看
- 支付状态同步
- 发货状态更新
- ERP 同步

## 5.5 支付配置

用户希望为不同站点配置支付通道，并查看支付成功率与异常订单。

预期效果：
- 绑定支付通道
- 配置回调
- 查看支付状态
- 处理退款与异常单

---

## 6. 功能范围

## 6.1 账号与租户模块

必须具备：
- 注册 / 登录
- JWT 鉴权
- 租户初始化
- 用户资料管理
- 员工邀请
- 角色权限
- 操作日志

业务规则：
- 每个租户至少有一个主账号
- 套餐决定可创建站点数、员工数、订单量或高级功能权限

## 6.2 一键建站模块

### 功能目标

通过模板快速创建站点。

### 用户输入

- 站点名称
- 子域名前缀
- 站点模板
- 目标地区
- 默认语言
- 默认币种
- 默认支付通道

### 系统处理

- 校验套餐额度
- 校验子域名唯一性
- 创建站点记录
- 调用底层建站引擎
- 初始化页面、区块、主题变量、基础设置

### 输出结果

- 站点创建成功或失败
- 站点访问地址
- 站点工作台入口

### 业务规则

- 子域名必须唯一
- 创建失败必须可查看失败原因
- 建站过程采用异步任务，前端展示进度状态

## 6.3 站点管理模块

### 功能

- 站点列表
- 站点搜索和筛选
- 站点状态查看
- 站点编辑
- 复制站点
- 归档 / 删除
- 进入工作台

### 站点状态

- CREATING
- ACTIVE
- FAILED
- DISABLED
- ARCHIVED

### 列表核心字段

- 站点名称
- 模板
- 访问域名
- 国家 / 语言 / 币种
- 支付状态
- 斗篷状态
- 最近订单量
- 创建时间

## 6.4 页面布局编辑模块

这是必须优先实现的核心模块。

### 功能

- 页面列表
- 区块拖拽排序
- 区块新增 / 删除
- 区块属性编辑
- 主题变量编辑
- 页头 / 页脚全局配置
- 草稿保存
- 发布上线
- 历史版本回滚

### 页面类型

- 首页
- 商品详情页
- 落地页
- 结账页
- 支付成功页
- 通用内容页

### 区块类型

- Banner
- 图文介绍
- 商品卡片
- 信任背书
- 评价
- FAQ
- 倒计时
- 表单 / 结账
- 页脚

### 业务规则

- 每次发布生成独立版本
- 已发布版本必须可回滚
- 页面编辑权限支持按站点控制

## 6.5 模板中心

### 功能

- 模板列表
- 模板预览
- 基于模板建站
- 模板复制
- 模板保存为私有模板

### 目标

缩短新站搭建时间，支持站群快速复制。

## 6.6 斗篷管理模块

### 功能

- 创建斗篷规则
- 规则排序
- 条件配置
- 命中结果配置
- 日志查看
- 开关控制
- 灰度发布

### 条件维度

- 国家
- IP 段
- ASN
- Referer
- UTM 参数
- 设备类型
- 浏览器
- 首次访问
- 白名单 / 黑名单

### 结果类型

- 展示正常页
- 展示审核页
- 跳转到外部页面
- 直接拒绝

### 非功能要求

- 规则执行要快
- 日志写入不能阻塞主请求

## 6.7 商品中心

### 功能

- 商品列表
- 商品创建与编辑
- SKU 管理
- 价格管理
- 库存管理
- 商品上下架

### 业务规则

- 商品可绑定多个站点
- 支持按站点定价扩展

## 6.8 订单中心

### 功能

- 订单列表
- 订单详情
- 支付状态查看
- 发货状态更新
- 客户信息查看
- 异常订单标记

### 订单状态建议

- CREATED
- PENDING_PAYMENT
- PAID
- SHIPPED
- COMPLETED
- CANCELED
- REFUNDED

## 6.9 ERP 模块

### 功能

- ERP 连接配置
- 商品同步
- 订单同步
- 库存同步
- 物流回传
- 同步日志

### 范围定义

- 本系统提供 ERP 中台能力
- 对复杂仓储和供应链流程通过外部 ERP 承接

## 6.10 支付中心

### 功能

- 支付通道管理
- 站点支付绑定
- 支付单创建
- 支付回调处理
- 支付日志
- 退款接口预留

### 关键要求

- 支付回调必须验签
- 支付单状态变化必须可追踪
- 每个站点可绑定一个或多个支付通道

## 6.11 域名与 SSL 模块

### 功能

- 默认子域名分配
- 自定义域名添加
- DNS 检测
- SSL 证书签发
- 续期状态监控

## 6.12 数据看板

### 功能

- 站点概览
- 订单趋势
- 支付成功率
- 访问来源
- 斗篷命中情况
- 异常告警

## 8. MVP 范围

第一阶段必须交付：
- 多租户账号体系
- 一键建站
- 站点列表与工作台
- 页面编辑器 MVP
- 域名与 SSL 基础能力
- 商品中心基础版
- 订单中心基础版
- 支付通道配置基础版
- 斗篷规则 MVP

第一阶段暂不强求：
- 复杂营销活动
- 高级 BI 分析
- 完整采购 / 供应链 ERP
- 多仓波次发货
- 自动化风控策略编排平台

---

## 9. 非功能需求

性能要求：
- 站点列表和订单列表支持分页
- 斗篷规则判定应保持低延迟
- 页面编辑保存应支持自动草稿

安全要求：
- 接口鉴权
- 租户隔离
- 关键配置审计
- 支付回调验签

可用性要求：
- 建站任务失败可重试
- 外部接口失败可补偿
- 页面发布失败不影响已上线版本

可维护性要求：
- 外部支付、ERP、DNS 都采用适配器方式集成
- 页面区块采用 schema 配置，减少硬编码页面开发

---

## 10. 验收标准

满足以下条件可认为系统方向完成切换：

- 用户可以通过后台完成一键建站
- 用户可以在后台编辑首页和落地页布局并发布
- 用户可以为站点配置至少一种支付方式
- 用户可以配置至少一组斗篷规则并查看命中日志
- 用户可以查看商品、订单和基础库存数据
- 用户可以将订单同步到至少一种 ERP 连接器

---

## 11. 重构优先级建议

P0：
- 一键建站
- 站点工作台
- 页面编辑器 MVP
- 支付基础能力

P1：
- 斗篷规则 MVP
- 商品订单中台
- ERP 连接器

P2：
- 模板市场
- 高级支付对账
- 高级斗篷策略
- 数据分析与自动化运营
