# 跨境电商建站 SaaS UI 设计文档

## 1. 文档目标

本文档用于定义新系统的后台 UI 方向。界面设计要服务于以下核心目标：
- 一键建站
- 可视化编辑网站布局
- 管理斗篷规则
- 管理商品、订单、ERP、支付

整体 UI 不再以“站点接入 + 发文后台”为中心，而是转为“跨境电商建站运营工作台”。

---

## 2. 设计原则

### 2.1 目标气质

- 专业，但不要像传统 ERP 那样笨重
- 高效率，适合运营人员连续处理多个站点
- 有明显的“建站平台”属性，而不是纯数据后台
- 页面编辑器要比普通表单页更具产品感

### 2.2 体验原则

- 建站入口始终明显
- 站点级操作路径短
- 高频模块优先：站点、页面、订单、支付、斗篷
- 表格与卡片混合，不强行全站统一单一形态
- 编辑器页面与管理页视觉分层明显

---

## 3. 视觉语言

## 3.1 风格方向

建议采用“浅底色 + 深文本 + 强对比主色”的运营平台风格。

关键词：
- 冷静
- 速度感
- 轻科技
- 高可读性

## 3.2 色彩系统

品牌主色建议：
- 深蓝 `#1E3A8A`
- 电光蓝 `#2563EB`
- 青蓝强调 `#06B6D4`

功能辅助色：
- 成功 `#16A34A`
- 警告 `#D97706`
- 危险 `#DC2626`
- 紫红强调 `#BE185D`

中性色：
- 页面背景 `#F5F7FB`
- 卡片背景 `#FFFFFF`
- 一级文字 `#0F172A`
- 二级文字 `#475569`
- 边框 `#E2E8F0`

模块识别色建议：
- 建站：蓝色
- 编辑器：青色
- 斗篷：紫红
- 支付：绿色
- ERP / 订单：橙色

## 3.3 字体与层级

- 页面主标题：28px / 700
- 区块标题：18-20px / 600
- 表格正文：14px / 400
- 说明文字：12-13px / 400
- 数据大盘数值：30-36px / 700

## 3.4 圆角与阴影

- 页面卡片：20px
- 输入与按钮：12px
- 弹窗：20px
- 阴影偏轻，避免厚重 SaaS 模板感

---

## 4. 信息架构

## 4.1 一级导航

建议采用左侧主导航 + 顶部站点上下文栏。

一级导航建议：
- 工作台
- 站点
- 页面装修
- 商品
- 订单
- 斗篷
- 支付
- ERP
- 域名与设置

辅助入口：
- 模板中心
- 员工与权限
- 套餐与账单

## 4.2 顶部栏

固定展示：
- 当前站点切换器
- 快速建站按钮
- 全局搜索
- 通知中心
- 当前用户菜单

## 4.3 站点上下文切换

设计要求：
- 在大多数业务页面中，站点是最重要上下文
- 顶部必须能快速切换当前站点
- 切换后，页面、斗篷、支付、订单都自动切换上下文

---

## 5. 核心页面设计

## 5.1 工作台首页

### 页面目标

让商户一进入系统就能看到站点运营状态，并快速发起建站。

### 页面结构

1. 顶部欢迎区
- 今日核心数据
- 快速建站按钮
- 最近异常提醒

2. 核心指标卡
- 站点数
- 今日订单数
- 今日支付金额
- 支付成功率
- 斗篷命中率

3. 趋势区
- 近 7 日订单趋势
- 近 7 日支付趋势
- 异常告警列表

4. 最近活动
- 最近建站记录
- 最近发布页面
- 最近支付异常

### 设计要求

- 顶部要有明显的行动入口
- 数据卡不要过多，控制在 4-6 个
- 异常信息优先级高于普通动态

## 5.2 站点列表页

这是最核心的运营入口之一。

### 页面结构

1. 页面头部
- 标题
- 站点总数
- 快速建站按钮

2. 筛选栏
- 搜索站点
- 按模板筛选
- 按状态筛选
- 按地区筛选

3. 站点列表区
- 推荐采用“卡片 + 表格”两种视图切换

### 站点卡片信息

- 站点封面缩略图
- 站点名称
- 模板标签
- 地区 / 语言 / 币种
- 当前域名
- 支付状态
- 斗篷状态
- 最近订单数

### 快捷操作

- 进入工作台
- 页面装修
- 斗篷设置
- 支付设置
- 域名设置

### 视觉方向

- 卡片带轻预览感
- 状态标签清晰，不要堆太多按钮
- 高优先操作固定露出

## 5.3 一键建站页

这是必须重点设计的页面。

### 页面目标

用最少字段完成建站，同时给用户足够的信心知道系统会自动完成什么。

### 页面结构

1. 左侧模板选择
- 模板卡片
- 模板预览图
- 模板标签

2. 右侧建站表单
- 站点名称
- 子域名前缀
- 国家 / 语言 / 币种
- 默认模板
- 默认支付方式

3. 底部结果说明
- 将自动创建的页面
- 将自动配置的基础能力
- 预估完成时间

### 交互要求

- 子域名实时校验
- 模板选择即时预览
- 提交后进入任务进度状态页

## 5.4 建站任务状态页

### 页面目标

展示建站进度和失败原因，避免用户对“正在建站”无感知。

### 展示内容

- 当前进度步骤
- 当前步骤状态
- 失败原因
- 重试按钮
- 建站成功后的工作台入口

## 5.5 页面装修编辑器

这是本系统区别于普通站点管理后台的关键页面。

### 页面结构

1. 顶部编辑栏
- 页面名称
- 预览
- 保存草稿
- 发布
- 版本记录

2. 左侧面板
- 页面树
- 区块库
- 模板区块

3. 中间画布
- 页面可视化预览
- 选中区块高亮
- 拖拽排序

4. 右侧属性面板
- 文案
- 图片
- 颜色
- 按钮
- 布局
- 数据绑定

### 核心体验要求

- 支持拖拽排序
- 支持区块复制
- 支持撤销与重做
- 支持 PC / Mobile 预览切换
- 发布动作必须明确且可回滚

### 视觉要求

- 编辑器和普通后台页面要明显区分
- 中间画布区域要足够大
- 属性面板不能过于拥挤

## 5.6 模板中心

### 页面结构

- 模板分类
- 模板预览卡片
- 使用模板建站
- 保存为我的模板

### 设计方向

- 更像素材中心，不像纯表格页
- 预览图和标签信息优先

## 5.7 斗篷管理页

### 页面目标

让运营人员能快速理解规则和当前命中情况，而不是面对一堆抽象字段。

### 页面结构

1. 顶部概览
- 已启用规则数
- 今日命中量
- 风险异常数

2. 规则列表
- 规则名称
- 命中条件摘要
- 命中结果
- 状态
- 更新时间

3. 右侧或弹层编辑
- 条件配置器
- 优先级
- 结果配置

### 交互要求

- 条件配置尽量采用可组合表单
- 命中结果必须可视化表达
- 规则启停要有确认

## 5.8 商品管理页

### 页面结构

- 商品搜索筛选栏
- 商品表格
- 商品编辑抽屉
- SKU 与库存子表

### 设计重点

- 商品信息密度要高
- 编辑动作尽量在抽屉完成，减少跳转

## 5.9 订单管理页

### 页面结构

- 顶部订单指标
- 筛选区
- 订单表格
- 订单详情侧边栏

### 重点字段

- 订单号
- 站点
- 客户
- 金额
- 支付状态
- 发货状态
- 来源渠道

## 5.10 支付配置页

### 页面结构

- 通道列表
- 通道状态卡
- 回调配置
- 错误日志

### 设计重点

- 风险提示明显
- 成功率、失败率可快速看到

## 5.11 ERP 连接页

### 页面结构

- 已连接 ERP 列表
- 连接配置表单
- 最近同步结果
- 错误日志

## 5.12 域名与 SSL 页

### 页面结构

- 当前主域名卡
- 域名列表
- 添加域名表单
- DNS 指引
- SSL 状态

---

## 6. 关键组件规范

## 6.1 数据卡

用于首页和模块概览。

规范：
- 一个卡片只强调一个主数值
- 带轻量趋势或状态说明
- 支持风险色提示

## 6.2 状态标签

必须统一：
- 建站状态
- 支付状态
- 斗篷状态
- 订单状态

建议：
- 成功类用绿
- 处理中用蓝
- 风险类用橙
- 失败类用红

## 6.3 区块卡片

用于页面编辑器中的区块库。

内容：
- 图标
- 名称
- 简短说明
- 预览示意

## 6.4 侧边抽屉

适合：
- 编辑商品
- 编辑订单备注
- 编辑支付配置

不适合：
- 页面可视化编辑
- 复杂斗篷规则编排

## 6.5 表格

要求：
- 支持固定表头
- 支持列排序
- 支持批量操作
- 行级操作收敛，不要过度堆叠

---

## 7. 交互规范

## 7.1 保存与发布

适用页面：
- 页面编辑器
- 斗篷规则
- 支付配置

规范：
- 保存草稿与发布上线分离
- 发布前显示影响范围
- 发布后有明确成功反馈

## 7.2 危险操作

包括：
- 删除站点
- 关闭支付通道
- 停用斗篷规则
- 回滚页面版本

规范：
- 二次确认
- 明确后果提示
- 记录操作日志

## 7.3 异步任务反馈

适用：
- 建站
- 域名检测
- ERP 同步

规范：
- 用户提交后立即看到状态页或状态条
- 要显示处理中、成功、失败、可重试

---

## 8. 响应式要求

桌面端为主，但必须支持移动端基础查看。

桌面端重点：
- 工作台
- 站点列表
- 页面编辑器
- 订单处理

移动端要求：
- 支持查看核心数据和订单
- 不要求完整页面编辑能力

---

## 9. 设计 Token 建议

建议统一定义：
- 颜色变量
- 间距变量
- 圆角变量
- 阴影变量
- 状态色变量
- 图表色板

这样后续站点后台和编辑器才能保持一致。

---

## 10. MVP UI 优先级

P0 页面：
- 工作台首页
- 站点列表页
- 一键建站页
- 建站任务状态页
- 页面装修编辑器

P1 页面：
- 斗篷管理页
- 商品管理页
- 订单管理页
- 支付配置页

P2 页面：
- ERP 连接页
- 模板中心
- 套餐账单页

---

## 11. 最终设计结论

本系统 UI 的设计重心必须从“后台录入数据”转向“运营驱动的建站工作台”。

最关键的三个页面是：
- 一键建站页
- 站点列表页
- 页面装修编辑器

如果这三个页面没有建立明确体验优势，系统很容易退化成普通管理后台，无法体现“跨境电商建站平台”的核心价值。
